# Contributing during the alpha

Thanks for trying TinyPilot Dashboard. During this alpha the most valuable thing you can do is **share feedback** — please open a [GitHub issue](https://github.com/tiny-pilot/tinypilot-dashboard/issues/new/choose).

## What we're collecting right now

- Bugs and unexpected behavior, with the details our [bug report template](.github/ISSUE_TEMPLATE/bug_report.md) asks for.
- Feedback on the install flow, the deployment posture (LAN / VPN / zero-trust overlay), and the device snapshot fields you'd want to see.
- Confirmation that the dashboard works for you and on which TinyPilot firmware versions.

## What we're not yet set up to handle

We're not accepting pull requests during the alpha. This release is a small, focused effort to gather customer feedback before the project is opened up. If you have a fix or feature you'd love to see, please describe it in an issue and we'll track it there — that's far more useful to us than a PR at this stage.

When the project moves out of alpha, we'll update this file with contribution guidelines and dev-loop notes.

## Reaching us

- Bugs and feature requests: [GitHub issues](https://github.com/tiny-pilot/tinypilot-dashboard/issues)
- TinyPilot product questions and support: [tinypilotkvm.com](https://tinypilotkvm.com)
